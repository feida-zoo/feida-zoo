## 四、跨成员通信机制

### 4.1 交付物流转规则

```
┌─────────────┐      写入 shared/      ┌─────────────┐
│   交付者     │ ────────────────────▶ │   接收者    │
│ (产出方)    │   【禁止Panda转发】     │  (消费方)   │
```

### 4.2 共享目录结构

```

├── alpha_architecture_v1.1.md    # 架构设计文档

```

## 五、跨成员交互铁则（第11条新增）

### 🛡️ 铁则 #11：交付物写入共享目录

> **所有跨角色交付的产出物必须写入共享目录，禁止通过Panda转发大段内容。**

**实施规范：**

1. **产出物类型**：架构设计文档、代码审查报告、设计稿说明、技术规范等
2. **写入路径**：`/home/afei/workspace/code/feida_zoo/framework/shared/`
3. **命名规范**：`{产出者}_{接收者}_{内容摘要}.md`
4. **禁止行为**：不得通过 Panda 主会话直接发送大段文本给接收者
5. **通知机制**：交付完成后，通知接收者前往 shared/ 目录读取

**例外场景**（可简短摘要通知）：
- 紧急变更需立即告知
- 交付物小于 200 字
- 接收者明确表示接受摘要模式
