## 概述

本需求核心是替换头像源文件 + 修复后端路径，**不涉及新增 UI 组件或交互流程**。UI 层面的变化是头像图片的视觉表现从「变形/尺寸不对」变为「正确的正方形裁剪」。

### 改前 vs 改后对比

| 区域 | 改前效果 | 改后效果 | 原因 |
|------|---------|---------|------|
| 成员 Tab | alpha 头像 1408×768，在 80×80 圆形中变形 | 1024×1024 完美圆形裁剪 | 源文件尺寸错误→替换为正方形 |
| 成员 Tab | duci 头像 1408×768，同上 | 1024×1024 完美圆形裁剪 | 同上 |
| 成员 Tab | panda 头像 512×512，圆形但分辨率低 | 1024×1024 更清晰 | 换高清源 |
| 统计 Tab | panda 头像 `/avatar/panda` → 404 → 显示 emoji fallback | 正确显示 PNG 头像 | `_serve_avatar()` 路径修复 |
| 看板卡片 | duci 头像使用 `stinger.png` 映射 | 直接使用 `duci.png`（相同文件） | 去除硬编码 |

## 视觉说明

### 裁剪效果

CSS 裁剪参数（三个区域统一行为）：

| CSS 属性 | 值 | 效果 |
|---------|-----|------|
| `object-fit` | `cover` | 保持宽高比，填满容器 |
| `border-radius` | `50%` | 圆形裁剪 |

改前：1408×768 → 中心裁出 768×768 正方形 → 缩放到 80×80 → 人物头像靠左偏移
改后：1024×1024 → 缩放到 80×80 → 完美居中

### 各区域 avatar 显示参数

| 区域 | 容器尺寸 | 边框 | 用途 |
|------|---------|------|------|
| 成员 Tab 卡片 | 80×80 | 3px `rgba(137,180,250,0.3)` hover变蓝 | 大尺寸展示 |
| 统计 Tab 成员状态 | 50×50 | 2px `rgba(255,255,255,0.4)` | 中等尺寸展示 |
| 看板卡片 assignee | 24×24 | 1px `var(--border-color)` | 紧凑展示 |
