# P1 技术设计方案：聊天室界面无数据显示

## 问题分析

### 根因
1. **CSS 特异性 Bug**：`#tab-chat.tab-content` 的 `display: flex` 覆盖了 `.tab-content { display: none }`，导致聊天室内容在所有 tab 下始终可见，但因为没有 `.active` 类，内部布局异常（已在 a3ef42c 修复 ✅ 毒刺审计通过）
2. **ZooMesh 守护进程未运行**：聊天室数据来自 ZooMesh HTTP API，ZooMesh 不运行时 `/api/chat` 返回空数组（已在启动 ZooMesh 后恢复）
3. **没有加载态显示**：用户切换聊天室 tab 时，API 请求期间页面空白，无任何反馈

### 结论
当前代码中 Chat API 数据流正常（50 条消息），渲染逻辑正常，CSS 已修复。

## 改动范围
仅增强 `loadChat()` 的 UI 状态处理（代码已写好，之前错误提交时包含但被回滚，需重新提交）：
- 加载中：显示 spinner + "正在加载..."
- 空数据：显示 "暂无聊天消息"
- 加载失败：显示 "加载失败" + 重试按钮
