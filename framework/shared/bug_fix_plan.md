# Bug 修复计划 - Task 1.4 安全删除机制

## P1 致命漏洞修复

### 1. 并发竞争条件
**问题**: `_read_log()` 和 `_write_log()` 方法存在竞态条件
**修复方案**:
- 添加文件锁机制（fcntl 或 threading.Lock）
- 使用原子读写操作

### 2. 路径遍历攻击
**问题**: 未对输入的 `path` 参数进行规范化验证和边界检查
**修复方案**:
- 添加 `_validate_path()` 方法
- 使用 `resolve()` 和 `relative_to()` 验证路径在工作区范围内

## P2 重要漏洞修复

### 3. 日志文件无限增长
**问题**: 日志文件会无限期增长，没有自动清理机制
**修复方案**:
- 添加最大日志条目数限制（如 10000 条）
- 定期清理旧日志

### 4. 静默日志写入失败
**问题**: 日志写入失败时静默忽略
**修复方案**:
- 移除静默忽略，改为抛出异常或至少记录到 stderr
- 统一异常处理策略

### 5. 相对路径解析漏洞
**问题**: `_get_trash_path()` 未处理路径不在工作区的情况
**修复方案**:
- 使用 `_validate_path()` 验证路径
- 确保路径在工作区内

### 6. 异常处理不一致
**问题**: 异常处理策略不统一
**修复方案**:
- 统一所有方法的异常处理
- 要么全部重新抛出并记录，要么提供一致的处理方式
