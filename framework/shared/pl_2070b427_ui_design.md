## 交互逻辑

### 核心方案（后端过滤 — 无前端交互改动）

| 操作 | 改前行为 | 改后行为 | 改动 |
|------|---------|---------|------|
| 切换到成员管理 Tab | 加载 6 张卡片 | 加载 3 张卡片 | 无（数据量自动减少） |
| 每小时自动刷新 | 重新请求 `/api/members` | 同左，但只收到 3 条 | 无 |
| SSE 实时状态推送 | 更新 6 张卡片的状态指示器 | 仅更新 3 张 | 无 |

### 方案 D（非活跃折叠区 — 可选）

| 操作 | 行为 |
|------|------|
| 点击「⏵ 非活跃成员」 | 折叠区展开，显示 3 张非活跃成员卡片 |
| 再次点击「⏷ 非活跃成员」 | 折叠区收起 |
| 非活跃成员卡片 hover | 与活跃卡片相同 hover 效果，但卡片整体略微灰化（0.85 opacity） |
| 每次 `/api/members` 加载 | 后端返回 `is_active` 字段，前端按值分两组渲染 |

## 状态定义

### 唯一状态：渲染结果

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| **加载中** | 首次进入 Tab / 自动刷新 | `loading` 旋转动画 +「正在加载成员数据...」文字 |
| **渲染成功（有活跃成员）** | API 返回含 1+ 活跃成员 | Grid 布局展示成员卡片 |
| **渲染成功（0 活跃成员）** | API 返回空数组 | 空状态插画 +「暂无成员数据」提示 |
| **加载失败** | 网络 / HTTP 错误 | 红色错误图标 +「加载成员数据失败」文字 |

### 成员卡片状态（已有，不变）

| 状态 | CSS class | 指示器 | 文字 |
|------|-----------|--------|------|
| executing / online | `.online` | 🟢 绿色圆点 | 🟢 在线 |
| idle / sleeping | `.idle` | 🟡 黄色圆点 | 🟢 空闲 / 💤 休眠 |
| dead / terminated / unknown | `.offline` | 🔴 红色圆点 | 🔴 离线 / ⚫ 离线 |

非活跃成员的卡片（方案 D）独立于上述状态体系，固定为灰化展示 +「已退出」标签。

## 视觉说明

### 非活跃卡片样式（方案 D）

- 保持与活跃卡片相同的 HTML 结构（reuse `member-tab-card`）
- 额外 class: `member-tab-card--inactive`
- 视觉差异：

| 属性 | 活跃卡片 | 非活跃卡片 |
|------|---------|-----------|
| opacity | 1.0 | 0.7 |
| 角色徽章 | 正常角色名 | 「已退出」(灰色) |
| 模型名 | 显示 | 不显示（或显示「—」） |
| 状态指示圆点 | 显示 | 显示为灰色空心圆 |
| 背景色 | `--card-bg` | 同色但半透明 |
| 底部边框 | 无 | 浅灰色虚线 |

### CSS 新增（方案 D）

```css
.member-tab-card--inactive {
    opacity: 0.7;
    filter: grayscale(30%);
}
.member-tab-card--inactive .member-tab-role-badge {

}

    display: none;  /* hidden by default */
}

}
```
